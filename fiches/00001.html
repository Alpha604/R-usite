Définition et importance

Qu'est-ce qu'un algorithme ?

Un algorithme est une suite finie et non ambiguë d'instructions permettant de résoudre un problème ou d'obtenir un résultat. Les algorithmes sont au cœur de l'informatique et constituent la base de tous les programmes.

L'étude des algorithmes permet de déterminer leur efficacité en termes de temps d'exécution et d'utilisation de la mémoire. Cette analyse est cruciale pour le développement de logiciels performants.

Complexité algorithmique

La notation Big O (O) permet d'exprimer la complexité temporelle d'un algorithme en fonction de la taille de ses données d'entrée (n).

function rechercheLineaire(tableau, cible) {
    for (let i = 0; i < tableau.length; i++) {
        if (tableau[i] === cible) {
            return i;
        }
    }
    return -1;
}

Complexité : O(n) dans le pire des cas

Complexité Notation Exemple Efficacité
Constante O(1) Accès à un élément de tableau Excellente
Logarithmique O(log n) Recherche binaire Très bonne
Linéaire O(n) Recherche linéaire Bonne
Quadratique O(n²) Tri bulle Médiocre

Structures de données

Les structures fondamentales

Les structures de données permettent d'organiser et de stocker des données de manière efficace. Le choix d'une structure adaptée est essentiel pour optimiser les performances d'un algorithme.

Les principales structures de données incluent :

  • Tableaux : Accès rapide aux éléments, taille fixe
  • Listes chaînées : Insertion/suppression efficaces, taille dynamique
  • Piles : LIFO (Last In, First Out)
  • Files : FIFO (First In, First Out)
  • Arbres : Hiérarchie, recherche rapide
  • Graphes : Relations complexes entre éléments

Exercices pratiques

1. Quel est le temps d'exécution (complexité) de l'algorithme suivant ?

function sommeTableau(tableau) {
    let somme = 0;
    for (let i = 0; i < tableau.length; i++) {
        somme += tableau[i];
    }
    return somme;
}

Réponse : O(n) - L'algorithme parcourt chaque élément du tableau une fois, donc le temps d'exécution est proportionnel à la taille du tableau (n).

2. Proposez un algorithme pour trouver le plus grand élément dans un tableau d'entiers. Quelle est sa complexité ?

Réponse :

function plusGrandElement(tableau) {
    if (tableau.length === 0) return null;
    
    let max = tableau[0];
    for (let i = 1; i < tableau.length; i++) {
        if (tableau[i] > max) {
            max = tableau[i];
        }
    }
    return max;
}

Complexité : O(n) - L'algorithme examine chaque élément du tableau une fois.

3. Écrivez un algorithme récursif pour calculer la factorielle d'un nombre n.

Réponse :

function factorielle(n) {
    if (n <= 1) {
        return 1;
    }
    return n * factorielle(n - 1);
}

Complexité : O(n) - L'algorithme effectue n appels récursifs.
